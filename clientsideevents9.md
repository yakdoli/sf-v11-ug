ClientSideOnNodeSelect

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

In the above example, the **ClientSideOnNodeSelect** property is set to the \'nodeSelected\' client-side javascript function. This will get executed whenever a node is selected and passes the selected node as a parameter to the **NodeSelected** event.

ClientSideOnNodeRename

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

In the above example, **ClientSideOnNodeRename** property is set to the \'nodeRenamed\' client-side javascript function. This will get executed whenever a node is renamed and passes the renamed node as parameters to the **NodeRenamed** event. Make sure that the **EditNode** property is set to **True**, to allow  node to be edited. You can get the nodes old text using the renamed node which is passed as a parameter to the NodeRenamed event.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

ClientSideOnNodeMove

[]{style="FONT-SIZE: 8pt"} 

In the above example, **ClientSideOnNodeMove** property is set to the \'nodeMoved\' client-side javascript function. This will get executed whenever a node is dropped and passes the dropped node as parameter to the **NodeMoved** event. Make sure that the **DragAndDropEnabled** property is set to **True**, to allow nodes to be moved.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

ClientSideOnNodeDrop

[]{style="FONT-SIZE: 8pt"} 

In the above example, **ClientSideOnNodeDrop** property is set to the \'nodeDropped\' client-side javascript function. This will get executed whenever a node is dropped and passes the dropped node as parameters to the **NodeDropped** event. Make sure that the **DragAndDropEnabled** property is set to **True**, to allow node to be dropped.

 

**node.SourceNode.Text** and **node.TartgetNode.Text** can be used to obtain the source and the target nodes of the dropped node.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

ClientSideOnNodeExpand

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

In the above example, the **ClientSideOnNodeExpand** property is set to the \'nodeExpanded\' client-side javascript function. This will get executed whenever a node gets expanded and passes the expanded node as a parameter to the **NodeExpanded** event.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

ClientSideOnNodeCollapse

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

In the above example, the **ClientSideOnNodeCollapse** property is set to the \'nodeCollapsed\' client-side javascript function. This will get executed whenever a node gets collapsed and passes the collapsed node as a parameter to the **NodeCollapsed** event.

[]{style="FONT-SIZE: 8pt"} 

ClientSideOnNodeCheckChanged

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

In the above example, **ClientSideOnNodeCheckChanged** property is set to the \'nodeChecked\' client-side javascript function. This will get executed whenever a node in the checkbox is checked / unchecked. Here **Checked** attribute gives the status of the checkbox, whether it is checked / unchecked.

###### 5.3.1.3.2.6 ClientSideOnNodeDrop event {#clientsideonnodedrop-event style="tab-stops: 0pt"}

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

**ClientSideOnNodeDrop** event is fired on node drop. Parameter of this event contains the following properties.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

[·      ]{style="FONT-FAMILY: Symbol"}**SourceNode**: dropping client side Treeview node object

[·      ]{style="FONT-FAMILY: Symbol"}**TargetNode**: target client side Treeview node object for dropping item

[·      ]{style="FONT-FAMILY: Symbol"}**Event**: browser event object

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

Make sure that the **DragDropEnabled** property is set to **True** to allow nodes to be moved. **DroppingAccrossTreesEnabled** is set **True** to allow nodes to be dropped from another Treeview, **DropSiblingEnabled** property indicates whether to allow dropping to create siblings (drop between nodes). This event can be canceled by simply returning False from the event handler and node dropping will be canceled.

###### 5.3.1.3.2.7 ClientSideOnNodeMove event {#clientsideonnodemove-event style="tab-stops: 0pt"}

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

**ClientSideOnNodeMove** event is fired on node is moved. Make sure that the **DragDropEnabled** property is set to **True** to allow nodes to be moved and **DraggingAccrossTreesEnabled** is set **True** to allow nodes to be dragged to another Treeview. Parameter of this event is **ClientSideData** object.
