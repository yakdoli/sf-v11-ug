### Collection Base {#collection-base style="tab-stops: 0pt"}

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

The implementation of a custom collection through the CollectionBase is simpler as it is derived from the IList.

 

Any class that inherits the CollectionBase abstract class can serve as a data source for the GGC. The below code snippet will create a CustomerCollection class which is derived from CollectionBase abstract class. It is a strongly-typed collection that holds a number of Customer objects. Public properties of Customer class will be represented as fields in GGC. You should define or override the required properties and methods of CollectionBase abstract class such as Add, IndexOf, and so on.

 

When CustomerCollection class is ready, you should simply create an instance of it and add some instances of Customer class to this instance through the Add method. Then, we assign the instance of CustomerCollection class as datasource for the GGC on Page_Load. GGC will step through public properties of Customer class and these properties will be represented as fields in the GGC.
