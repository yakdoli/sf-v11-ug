## DomainGridDataControl    {#domaingriddatacontrol style="tab-stops: 0pt"}

**[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"}** 

**DomainGridDataControl** lets users to get data form **DomainDataSource**. **DomainGridDataControl** depends on **GridDataControl** and all the properties in **GridDataControl** are supported in **DomainGridDataControl** except **Nested Grids. ** This control is useful while developing a **RIA** application which uses **DomianDataSource** as input collection.

**[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"}** 

Prerequisites

  -------------------------- --------------------------------
  Development Environments    Silverlight
  .NET Framework versions     3.5, 4.0
  Mandatory installation      WCF RIA services 2008 or 2010
  -------------------------- --------------------------------

**[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b"}** 

Creating a DomainGridDataControl Application

1.   Create a Silverlight application with RIA service enabled and with an ASP.NET web project.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

[]{style="FONT-FAMILY: 'Calibri','sans-serif'; COLOR: black"} 

![](ImagesExt/image61_23.jpg){border="0"}

Figure 12: Silverlight Application

*[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"}* 

2.   Add a **Database** file to the web project.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

\
![](ImagesExt/image61_24.jpg){border="0"}

Figure 13: Added Database File to the Web Project

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

3.   Add a Data Model to your application. It can be a **LINQ** to **SQL** data model, an Entity Framework Data Model, or any other data model.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

![](ImagesExt/image61_25.jpg){border="0"}

Figure 14: Added Data Model

 

*[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"}* 

![](ImagesExt/image61_26.jpg){border="0"}

Figure 15: LINQ to SQL Classes

***[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"}*** 

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

4.   Add a new **Domain Service**.

5.   Add a new Item to the web project, and select the **Domain Service** template.

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"} 

![](ImagesExt/image61_27.jpg){border="0"}

Figure 16: Domain Service Class

[]{style="FONT-FAMILY: 'Calibri','sans-serif'; COLOR: black"} 

[]{style="FONT-FAMILY: 'Calibri','sans-serif'; COLOR: black"} 

6.   After you add **Domain Service**, the New **Domain Service** Class Dialog is shown.

7.   Select the Data Context, the entities you want to expose and whether you want to allow editing.

8.   Click **OK**.

![](ImagesExt/image61_28.jpg){border="0"}

Figure 17: New Domain Service

***[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"}*** 

***[]{style="FONT-FAMILY: 'Trebuchet MS','sans-serif'; COLOR: #15428b; FONT-SIZE: 9pt"}*** 

 

9.   In the Silverlight application add references to System.Windows.Controls.Data and to RIA related assemblies.

10.  Add a DomainDataSource control in XAML.

11.  Drag and drop DomainDataGrid into the designer surface. 

12.  Set RIASource property to DomainDatasource.

Feature Summary

**Sorting** -- Click on the column header to trigger **Sorting**.

::: {style="BORDER-BOTTOM: windowtext 1pt solid; BORDER-LEFT: medium none; PADDING-BOTTOM: 1pt; MARGIN-TOP: 9pt; PADDING-LEFT: 0pt; PADDING-RIGHT: 0pt; MARGIN-BOTTOM: 9pt; BORDER-TOP: windowtext 1pt solid; BORDER-RIGHT: medium none; PADDING-TOP: 1pt"}
![](ImagesExt/image61_30.png){border="0"}Note: Sorting happens in the server side and the data is retrieved and populated in the client side.
:::

 

**Filtering** -- Enter appropriate data in the **Filter pane** to trigger **Filtering**.

::: {style="BORDER-BOTTOM: windowtext 1pt solid; BORDER-LEFT: medium none; PADDING-BOTTOM: 1pt; MARGIN-TOP: 9pt; PADDING-LEFT: 0pt; PADDING-RIGHT: 0pt; MARGIN-BOTTOM: 9pt; BORDER-TOP: windowtext 1pt solid; BORDER-RIGHT: medium none; PADDING-TOP: 1pt"}
![](ImagesExt/image61_30.png){border="0"}Note: Filtering also occurs on the server side and the data is populated on the client side.
:::

**Filter pane** is disabled immediately after entering appropriate data in the **Filter pane** till the data gets populated.

 

**Grouping** - Dragging and Dropping columns in **GroupDropArea** triggers **Grouping**.

::: {style="BORDER-BOTTOM: windowtext 1pt solid; BORDER-LEFT: medium none; PADDING-BOTTOM: 1pt; MARGIN-TOP: 9pt; PADDING-LEFT: 0pt; PADDING-RIGHT: 0pt; MARGIN-BOTTOM: 9pt; BORDER-TOP: windowtext 1pt solid; BORDER-RIGHT: medium none; PADDING-TOP: 1pt"}
![](ImagesExt/image61_30.png){border="0"}Note: Grouping is based on client side data.
:::

 

 **Paging -** Paging is supported by adding a **DataPager** control in **XAML** as shown in above sample XAML code.

![](ImagesExt/image61_31.png){border="0"}Note: Paging occurs in the server side and the data is populated in the client side.
