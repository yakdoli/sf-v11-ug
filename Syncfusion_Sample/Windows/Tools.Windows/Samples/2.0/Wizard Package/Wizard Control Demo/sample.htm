Wizard Control Demo

This sample illustrates how the Wizard control can be used to create full-fledged Wizards.

Features:

Page Selection

Usually, events occur before and after button clicks and after a page is selected.

The events fired during page selection are listed below in the order in which they are fired.

  • The ValidatePage event occurs if the Next or Finish button is clicked. This is a good place to validate whether the user has provided valid information before moving to the next step. If canceled, further events will not occur.


  • The BeforeBack, BeforeNext, and BeforeFinish events occur before the corresponding action happens. If canceled, further events will not occur. These events provide you with more ways to validate pages.


  • A Click event, NextClick for example, is used to notify you regarding button clicks.


  • The BeforePageSelected event has a reference to the new page that will be selected in the event handler, if you can specify a different page to be selected in the event arguments.


  • The AfterPageSelected event is used to notify you when a new page has been selected.


  • The newly selected WizardControlPage fires its PageLoad event to notify you that a page has been loaded.


  • Back, Next, Finish, Cancel, or Help events occur with respect to button clicks.
