XP TabControlAdv

This sample illustrates the design-time setup of the TabControlAdv class.

XP tabs are a fully native, .NET tab-control implementation that comes with more configuration and customization capabilities than the legacy wrapper of the .NET framework. It provides all the basic functionalities of a standard tab control while offering much more.

Features

  • Tab Styles

    • The most important characteristic of the tab control is the appearance of its tabs in the tab strip.

    There are multiple default implementations that come with the control: Office 2007, Tab Renderer 2-D, Tab Renderer 3-D, Internet Explorer 7, Tab Renderer Whidbey, Tab Renderer Office 2003, Tab Renderer Workbook Mode, Tab Renderer Docking Whidbey, One Note Style Renderer, and One Note Style Flat Tabs Renderer. Also, users can provide owner-drawn tabs.

  • Tab Primitives

    • TabControlAdv provides an easy way to navigate through tabs. By setting tab primitives (previously called navigation control), users can scroll to the first or last tab easily. Navigation control is more flexible; you can change the image on the buttons, show or hide certain buttons, and even cancel the navigation and use it as an Add or Remove tab.

  • Close Button

    • Tab primitives provide a close-button that is used to close active tab pages.

  • Close Button Like Internet Explorer 7

    • The tab control can have close buttons, similar to those in Internet Explorer 7, for all the tab pages. Also, the tab close button can be restricted to appear only for active tab pages.

  • Tab List Button

    • Tab primitives provide a drop-down button that lists tab pages. This avoids having to scroll through tab items to select a tab page.

  • Alignment and Sizing

    • A tab strip can be aligned to the top, left, right, or bottom of the control, and text alignment can be changed accordingly using the RotateTextWhenVertical property. Tabs can also be set to single-line or multi-line mode. The sizing mode can be set to either Normal, Fixed, Shrink to Fit, or Fill to Right.

  • Run Time Label Edit

    • A tab page's text can be edited during run time by using the LabelEdit property. The Label Edit feature has the following events associated with it.

  • BeforeEdit - Occurs when text enters the edit mode
  • AfterEdit - Occurs after new text is entered
  • LabelEditTextChanged - Occurs if the tab's text is changed
  • LabelEditChanged - Occurs when the Label Edit property is toggled

  • Image support

    • A tab page's text can be associated with images. Images can be aligned according to the alignment of the tabs.

    It is also possible to show and resize the bitmap and animated image in tab page using Image and ImageSize property of TabPageAdv.

  • Color Properties

    • TabControlAdv allows users to set different colors for active and inactive tabs using the ActiveTabColor, InactiveTabColor, TabBackColor, TabForeColor, and TabPanelBackColor properties.

  • Themes Support

    • TabControlAdv provides complete theme support.
