Docking Package

The Docking Package consists of the Essential Tools docking windows architecture that allows users to add Visual Studio.NET type dockable controls to Windows Forms applications. Any control can be set as a docking window and docking layouts can be hosted within forms, user controls, or any derivative of the Container Control type.

The framework supports a complete range of docking behaviors such as docking-floating, nested levels, tabbed groups, state transitions, AutoHide, MDI child transitions, non-dockable-non-floatable options, a unique fill mode, full state persistence, Office 2007-theme support, Office 2003 support, dock tab and dock label alignment, and VS 2005-style docking.

An advanced designer allows users to harness full WYSIWYG configuration of the dock layout.

The different samples included in Docking Package are listed below:

  • Simple Code - This sample shows you how to initialize and set up a docking layout using the docking manager's programmatic API.
  • SDI Demo - This sample demonstrates the docking layout in a standard form-type application.
  • MDI Demo - This sample demonstrates the behavior of the docking architecture in a MDI environment.
  • Window Fill - This sample mainly demonstrates the usage of the properties DockToFill and DisallowFloating. It also describes the CloseEnabled property of the docking manager.
  • Drag Provider Style - This sample illustrates different styles of drag providers, which will come into play when you drag a docked window.
  • Linked Managers - The Linked Manager concept allows us to transfer docked windows among two or more forms or user controls.
  • Nested Docking Layouts - This sample illustrates the implementation of nested docking layouts using user controls.
  • Custom Caption - This sample shows you how to add custom caption buttons to the docked controls and how to customize the caption bar.
