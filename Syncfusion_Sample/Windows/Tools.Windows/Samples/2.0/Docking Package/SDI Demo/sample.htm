SDI Demo

This sample demonstrates the docking layout in a standard form-type application.

The docking manager is hosted on a form and the docked controls share the form's client surface with the other controls.

This screenshot shows the typical display of docked controls.

Features:

  • This sample shows you how to use the DockingClientPanel class to provide a static region for the form's non-dockable controls. It also features the docking manager's powerful event-based customization model.

  • In this sample, all the dockable controls are set using the DockingManager.SetEnableDocking Extended property.

Docking Client Panel

The DockingClientPanel class is intended to serve as a container for the non-dockable controls on non-MDI Container forms. When using docking windows in the non-MDI case, all non-dockable controls originally intended to be located on the form should instead be placed on the docking client panel.

  • The Docking Windows framework repositions or resizes the docking client panel in tandem with changes in the host form’s client area during docking-undocking operations; thus, it provides a static surface for hosting the form’s non-dockable controls.

  • The DockingClientPanel.SizeToFit property is set to True. Turning on the SizeToFit property will force the docking client panel to start interacting with the Essential Tools docking architecture, and the control will automatically be resized-repositioned to occupy the form's client bounds that are left unoccupied by the docking windows.

Tab Control in the sample

  • On the docking client panel, a tab control with four TabPageAdv controls is hosted, which in turn hosts other controls.

  • The DockLabelAlignment style property changes the docking caption alignment. The DockTabAlignment style property changes the position of the docking windows tabs.

  • The DragProviderStyle property changes the dragging style of the docking windows. There are two styles: Standard and VS 2005.

  • The DockingManager provides functionality to freeze-unfreeze the docking windows. The SetFreezeResize (controlname, true / false) is used for this purpose.

  • EventLogs Tab - This tab hosts a list box that displays the sequence for firing various events.
