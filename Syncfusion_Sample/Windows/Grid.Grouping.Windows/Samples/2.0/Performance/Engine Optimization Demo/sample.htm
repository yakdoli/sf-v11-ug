Engine Optimization Demo

This sample demonstrates virtual-mode and without-counter-mode optimization.

Features:

  • Based on the schema specified, the engine determines whether specific optimizations may be applied.

  • If records are not sorted for a flat table, then the Virtual mode is applied.

  • Records do not have to be touched (except when drawing).

Interactive Features:

  • You will be able to see the automatic switching between the virtual mode and without-counter mode when you select the "Flat List VirtualMode" example.
  • The engine will be initialized using the virtual mode.
  • Once you sort by a column, the virtual mode will be turned off, but the without-counter mode will still continue to work.
  • Once you sort by a column, the counters will then be enabled.
