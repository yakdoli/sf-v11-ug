Localization Demo

Essential Diagram lets you customize built-in dialogs for a specific culture. This involves building custom-resource .dll files for specific cultures, as explained in this online help topic: Localization Support.

This screenshot shows the typical display for a localized Header-Footer dialog.

Note: The German translation is executed based on tools found online; Syncfusion does not guarantee the accuracy of the translation.

Features

This sample illustrates the localization of the various diagram dialogs such as the Header-Footer dialog, Page Borders dialog, Print Setup dialog, and Layout dialog. For example, in this sample, the diagram dialogs are localized for the German culture.

Note:

  • The German resource .dll files are copied to the bin folder in the following location: ..\bin\debug\de-DE.
  • The localizable property of the form is set to true.
  • To ensure that this sample application's thread runs in a German culture, even if the OS's regional settings are not set to German, you must explicitly set CurrentUICulture as follows (before the InitializeComponent() method): 

    • Thread.CurrentThread.CurrentUICulture = new System.Globalization.CultureInfo("de-DE");
