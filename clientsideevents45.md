#### Client-Side Events {#client-side-events style="tab-stops: 0pt"}

The **tree-view** control supports client-side event handling.

Events

 

 

  -------------------------------- --------------------------------------------------------------- ----------- ----------------
  Name                             Description                                                     Arguments   Reference Link
  ClientSideOnLoded                This event is raised immediately when the tree view is loaded   inst        \-
  ClientSideOnNodeClick            This event is raised when the end-user clicks a node.           args        \-
  ClientSideOnNodeExpand           This event is raised when the end-user expands a node.          args        \-
  ClientSideOnNodeCollapse         This event is raised when the end-user collapses a node.        args        \-
  ClientSideInlineEditValidation   This event is raised when the end-user edits a node.            Inst,args   \-
  -------------------------------- --------------------------------------------------------------- ----------- ----------------

*[[]{style="TEXT-DECORATION: none"}]{.underline}* 

Using Builder

 

The following steps describe how to handle client-side events raised by the tree view through the builder.

1.   In **View**, create a *ul-li* hierarchy of tree-view nodes and invoke the tree-view helper with the control ID as the first argument and the tree-view content ID as the second argument, followed by the **ClientSideOnLoded**, **ClientSideOnNodeClick**, **ClientSideOnNodeExpand**, **ClientSieOnNodeCollapse**, and **ClientSideInlineEditValidation** methods with the desired handlers as arguments.
